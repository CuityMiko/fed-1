# 前端知识体系的广度与深度

后台对于深度要求高，而前端对于广度要求高。

 - 用户体验设计

前端：用户喜爱的产品

基础 HTML + JS + CSS

广度 -> 用户体验、浏览器差异 -> 知识碎片 

jQuery + Backbone + xxx

后台：高并发高可用

纠结于 事务一致 锁 

Spring + Mybatis

数据库 + ORM + MVC

论技术上的深度，自然是后台更深

论用户体验上，自然是前端更深 两种很难衡量

常见问题：Ops 运维 -> 环境问题


对于前端来说，广度 First -> 深度。熟悉框架的相似之处，并且完成任务。

对于后端来说，选定语言，基本上就已经选好一切了，Java -> Spring，Scala -> Play，Ruby -> Rails
